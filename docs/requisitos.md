# 📋 Requisitos do Sistema — PrInt

## 📌 Requisitos Funcionais

O sistema deve:

- permitir cadastro de usuários  
- permitir login e autenticação  
- permitir entrada de sintomas  
- realizar perguntas adicionais  
- analisar respostas  
- classificar nível de urgência  
- exibir recomendações  
- armazenar histórico de atendimentos  
- permitir consulta ao histórico  

## 🔒 Requisitos Não Funcionais

- segurança dos dados (LGPD)  
- interface simples e intuitiva  
- boa performance  
- disponibilidade do sistema  
- escalabilidade  
- interface amigável  

## ⚠️ Restrições

- o sistema não deve fornecer diagnóstico médico  
- deve exibir avisos de orientação  
- deve proteger dados sensíveis  

## 📊 Critérios de Aceitação

- usuário consegue se cadastrar e acessar  
- sistema retorna classificação de urgência  
- histórico é salvo corretamente  
- interface é funcional e clara
